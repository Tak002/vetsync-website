# VetSync Help Center 기획서

## 설계 방향

- **참고 모델**: Crisp (심플 카드) + Linear (사이드바 + 검색) + Slack (자주 찾는 주제 태그)
- **구조**: 메인 페이지(8개 카테고리 카드 + 검색) → 카테고리 페이지(가이드 목록) → 개별 가이드 페이지
- **콘텐츠 형식**: 마크다운 기반 (추후 MDX 전환 가능), 스크린샷/GIF 첨부
- **i18n**: next-intl 연동 (한/영)
- **반응형**: 모바일 카드 풀너비, 데스크톱 3열 그리드

## 라우트 구조

```
/help                          → 메인 (카테고리 카드 + 검색 + 인기 가이드)
/help/[category]               → 카테고리별 가이드 목록
/help/[category]/[slug]        → 개별 가이드 페이지
```

## 기술 구현 메모

- **콘텐츠 소스**: `docs/help/` 마크다운 → 빌드 시 정적 페이지 생성 (또는 MDX)
- **검색**: 클라이언트 사이드 (Fuse.js 또는 FlexSearch) — 32개 가이드라 서버 불필요
- **이미지**: `public/help/` 디렉토리에 스크린샷/GIF 저장
- **SEO**: 각 가이드에 meta title/description
- **네비바**: 기존 Navbar에 "도움말" 링크 추가
